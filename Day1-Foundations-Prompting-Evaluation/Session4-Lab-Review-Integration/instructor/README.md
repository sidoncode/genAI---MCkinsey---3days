# Session 4 — Instructor Guide

## Day 1 Lab Review & Integration

### Facilitation Notes

**Duration:** 1 hour 30 minutes (3:30 – 5:00)

> **Note:** This is a facilitated discussion session — no notebook is required. Use these notes to guide the conversation.

### Suggested Timing

| Segment | Duration | Content |
|---|---|---|
| Recap | 20 min | Review key concepts from Sessions 1–3 |
| Integration Exercise | 25 min | Design a consulting AI system on paper |
| Group Discussion | 25 min | Share designs, compare approaches |
| Day 2 Preview | 15 min | LangChain, LangGraph, multi-agent overview |
| Q&A | 5 min | Open questions |

### Key Discussion Prompts

1. "If you were building an AI assistant for a McKinsey engagement team, which techniques from today would you use?"
2. "How would you evaluate the quality of AI-generated consulting deliverables for partner review?"
3. "What are the risks of using LLMs in client-facing work? How would you mitigate them?"
4. "Which embedding and reasoning patterns from Session 1 connect to the RAG pipeline we will build on Day 3?"

### Integration Exercise

Have participants sketch (on paper or whiteboard) an AI system for a McKinsey use case of their choice:
- What LLM capabilities does it need? (from Session 1)
- What prompting strategy would you use? (from Session 2)
- How would you evaluate its quality? (from Session 3)

### Bridge to Day 2

Preview what comes next:
- **Session 1 (Day 2):** Structured outputs — moving from free-form text to machine-readable JSON for consulting data extraction
- **Session 2 (Day 2):** LangChain — building composable analysis pipelines with custom consulting tools
- **Session 3 (Day 2):** LangGraph — stateful workflows that model consulting engagement lifecycles
- **Session 4 (Day 2):** Multi-agent — coordinating teams of specialized agents like a McKinsey engagement team
